👤 THÀNH VIÊN 1: ADMIN MODULE & CONTENT MANAGEMENT
* Thư mục làm việc: `Frontend/src/app/admin/` & Các file `*.controller.ts`, `*.service.ts` liên quan ở Backend.
* Nhiệm vụ:
  1. Quản lý nội dung (CRUD): Viết API và nối giao diện cho các bảng FAQ, Banner, Bài Viết.
  2. Xử lý Upload Ảnh: Bổ sung logic upload ảnh (lưu local file path hoặc dùng Cloudinary) cho Banner và Bài viết.
  3. Duyệt Voucher: Màn hình Voucher Approval. Gọi API lấy danh sách voucher có status 'PENDING_APPROVAL' (từ bên Partner đẩy sang) và làm nút Duyệt (đổi thành 'ACTIVE') / Từ chối ('REJECTED').
  4. Quản lý User: Xem danh sách, khóa tài khoản (chỉnh cờ isActive), xem lịch sử mua hàng.
  5. Thống kê (Dashboard): Viết hàm ORM Prisma (`aggregate`, `groupBy`) để thống kê doanh thu toàn hệ thống.

👤 THÀNH VIÊN 2: PARTNER MODULE & VOUCHER ENGINE
* Thư mục làm việc: `Frontend/src/app/partner/` & `voucher.controller.ts`, `voucher.service.ts`.
* Nhiệm vụ:
  1. Tạo Voucher & Save Draft: Làm form tạo voucher. Nếu ấn "Save Draft", gọi API lưu với `status: 'DRAFT'` (không ép buộc validate hết các trường). Nếu ấn "Submit", chuyển status thành `PENDING_APPROVAL` và gửi Admin chờ duyệt.
  2. Thuật toán sinh mã Voucher: Viết hàm sinh mã Voucher ngẫu nhiên (độ dài 10-12 ký tự, dùng `nanoid` ở Backend) khi một voucher được tạo và đẩy cho khách hàng.
  3. Thống kê (Dashboard): Viết hàm ORM Prisma thống kê số lượng voucher đã bán và doanh thu của riêng Partner đó.
  4. Quản lý chi nhánh: Viết CRUD cho các chi nhánh của đối tác.

👤 THÀNH VIÊN 3: CUSTOMER MODULE & CHECKOUT FLOW
* Thư mục làm việc: `Frontend/src/app/customer/` & `order.controller.ts`, `order.service.ts`.
* Nhiệm vụ:
  1. Storefront & Tìm kiếm: Hiển thị danh sách voucher đang 'ACTIVE', làm bộ lọc theo giá/category. Trang chi tiết voucher.
  2. Luồng Thanh toán (Orders): Giỏ hàng và Checkout. Khi thanh toán thành công, gọi API Backend để trừ 'stock' của Voucher và tạo record trong bảng Order.
  3. Đánh giá & Bình luận (Review): Chức năng để khách hàng để lại review sau khi sử dụng voucher.
  4. Customer Profile: Cập nhật thông tin cá nhân, xem lịch sử đơn hàng.

👤 THÀNH VIÊN 4 (LEAD/ARCHITECT): AUTH, LOGS & INTEGRATION
* Thư mục làm việc: `Frontend/src/app/auth/`, `schema.prisma`, `middlewares/`.
* Nhiệm vụ:
  1. Authentication Flow: Viết API Login/Register hoàn chỉnh. Dùng `bcrypt` mã hóa mật khẩu lưu vào DB. Dùng `zod` validate mật khẩu phức tạp (8 ký tự, chữ hoa, thường, số). Cấu hình trả về JWT token.
  2. Nhật ký hệ thống (System Logs): Viết một Service dùng chung để ghi Log. Test luồng log (ví dụ: bắt sự kiện Partner tạo voucher thì bắn log vào DB, hiển thị ở bảng Admin).
  3. Database Schema: Hoàn thiện `schema.prisma` từ file SQL cũ (thêm bảng Banner, FAQ...). Cập nhật và push DB DB chung (hoặc hướng dẫn team chạy migrate) để cả team đồng bộ.
  4. Code Review & Khớp nối: Giải quyết conflict Git nếu có, đảm bảo luồng liên kết giữa Admin và Partner (Duyệt voucher) chạy mượt mà.
